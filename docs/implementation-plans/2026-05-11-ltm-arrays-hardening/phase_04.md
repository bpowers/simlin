# LTM Arrays Hardening — Phase 4 Implementation Plan

**Goal:** Hoist *sliced* reducer subexpressions into aggregate nodes (not just whole-extent ones), and retire the last `Wildcard`-cross-product element-graph / link-score path. After Phase 4, every statically-describable inlined reducer — whole-extent or sliced (`SUM(pop[NYC,*])`, `SUM(matrix[D1,*])`) — is hoisted into an `$⁚ltm⁚agg⁚{n}` whose new `read_slice` descriptor records which rows it actually reads; the element graph routes only those rows through the agg (not a full cross-product), and `result_dims` carries the iterated axes' dims (so a partial-reduce subexpression becomes an arrayed agg). The one carve-out: a reducer over a *dynamic index* (`SUM(pop[idx,*])`, `idx` non-literal) stays unhoisted — a narrow, documented case, far smaller than today's "all slices stay conservative".

**Architecture:** `AggNode` gains `read_slice: Vec<AxisRead>` where `AxisRead ∈ {Pinned(elem), Iterated(dim), Reduced}` — one entry per source axis. `enumerate_agg_nodes`'s walker computes the read slice for each candidate reducer subexpression and hoists it iff every non-`Reduced` axis is `Pinned` or `Iterated` (statically describable); `result_dims` becomes the `Iterated` axes' dims (was always `vec![]` for synthetic aggs). `reducer_is_full_reduce` / `expr_is_full_extent` become "compute the read slice; full-extent = all-`Reduced`". The element graph (via the #520 IR's `ThroughAgg` routing) emits `source[<pinned>,<iterated>,<reduced→representative>] -> agg[<iterated>]` and `agg[<iterated>] -> target[e]` — only the read rows; the agg's `result_dims` drives the `agg -> target[e]` fan-out. The `Wildcard`-shape full-cross-product arm of `emit_edges_for_reference` (which currently handles `Wildcard | DynamicIndex` together) is narrowed to keep only the conservative path for the surviving carve-outs (`DynamicIndex`, and `SUM(pop[idx,*])` if it still classifies as `Wildcard`); per-element reducer link scores (`emit_source_to_agg_link_scores`) iterate only the read slice's elements; `link_score_dimensions` for a sliced-agg edge returns the `Iterated` dims; the agg-aux emission (`$⁚ltm⁚agg⁚{n}`, equation = the reducer subexpr) handles arrayed `result_dims` (the `Equation::ApplyToAll` arm already exists but was dead).

**Tech Stack:** Rust; the crate's `Expr2` AST + `IndexExpr2` (`src/ast/expr2.rs`); `BuiltinFn<Expr2>` (`src/builtins.rs`); salsa (`enumerate_agg_nodes` is `#[salsa::tracked(returns(ref))]`; `AggNode` derives `salsa::Update` — the new `read_slice`/`AxisRead` types must too); the #520 IR (`model_ltm_reference_sites`) and `ltm_agg::reducer_kind` from Phase 1; the iterated-dimension classification from Phase 3.

**Scope:** Phase 4 of 8 (GH #514). **Depends on Phase 1** (the #520 IR — `AggRef` points into `AggNodesResult.aggs`; the `ThroughAgg` routing consumes the read slice) and **Phase 3** (a sliced agg over an *iterated* dim reuses Phase 3's iterated-dimension classification and per-element-arrayed-target handling).

**Codebase verified:** 2026-05-12 (codebase-investigator report; see "Codebase notes" below).
